# Gerenciamento Global de Estados com Zustand

O **Zustand** é uma biblioteca de gerenciamento de estado global para React e React Native que se destaca por ser extremamente leve, rápida e minimalista. Diferente de soluções mais tradicionais e verbosas como o Redux, o Zustand foca na simplicidade, baseando-se fortemente no uso de React Hooks.

## Principais Vantagens

1. **Boilerplate Mínimo:** Não há necessidade de criar *actions*, *reducers*, *dispatchers* complexos ou envolver toda a sua aplicação com um `<Provider>`.
2. **Performance Otimizada:** O Zustand permite que os componentes se inscrevam (*subscribe*) apenas às partes específicas do estado que eles precisam. Isso evita re-renderizações desnecessárias quando outras partes do estado mudam.
3. **Curva de Aprendizado Suave:** Sua API é muito direta e intuitiva, facilitando a adoção para qualquer desenvolvedor que já compreenda o funcionamento básico dos React Hooks.
4. **Liberdade Arquitetural:** Ele não impõe uma estrutura rígida, dando a você a flexibilidade de organizar seus *stores* da maneira que melhor se adequar ao projeto (um único *store* global ou vários *stores* menores).

## Como Funciona na Prática?

Para utilizar o Zustand, você cria um *store* (que guarda tanto as variáveis de estado quanto as funções para atualizá-las) e ele retorna um Hook personalizado que você pode usar em qualquer componente.

### Exemplo Básico

**1. Criando o Store:**
```typescript
import { create } from 'zustand';

// Definindo a tipagem do estado
interface TaskState {
  tasks: string[];
  addTask: (task: string) => void;
  clearTasks: () => void;
}

// Criando o hook do store
export const useTaskStore = create<TaskState>((set) => ({
  tasks: [],
  addTask: (task) => set((state) => ({ tasks: [...state.tasks, task] })),
  clearTasks: () => set({ tasks: [] }),
}));
```

**2. Consumindo o Store em um Componente:**
```tsx
import React from 'react';
import { View, Text, Button } from 'react-native';
import { useTaskStore } from './store';

export default function TaskList() {
  // Acessando o estado e as ações diretamente
  const tasks = useTaskStore((state) => state.tasks);
  const addTask = useTaskStore((state) => state.addTask);

  return (
    <View>
      {tasks.map((task, index) => (
        <Text key={index}>{task}</Text>
      ))}
      <Button title="Adicionar Tarefa" onPress={() => addTask('Nova Tarefa')} />
    </View>
  );
}
```

## Conclusão

Para aplicações React Native e Expo modernas (como as estruturadas com Expo Router e o Novo Arquitetura habilitada), o Zustand é muitas vezes a escolha ideal para substituir estados locais complexos ou evitar o problema de *prop drilling*, mantendo a base de código limpa, manutenível e performática.
